# T0.1 Security Patches — Complete

## Files Changed
- `dashboard.js` — Full security patch (7 tasks)
- `index.html` — XSS esc(), auth headers, sessionCost fix

## Changes Made

### T0.1.1 — XSS: esc() function applied to all user data
- Added `esc()` function at line ~499 of index.html
- Applied to: cron names, schedule expressions, session targets, session IDs, model names, hostnames
- Total: 12+ interpolation points patched

### T0.1.2 — CORS: Same-origin restriction
- Replaced `Access-Control-Allow-Origin: *` with origin check
- Allows `http://localhost:18790` and `http://127.0.0.1:18790` only
- Added `Authorization` to allowed headers

### T0.1.3 — Auth: Bearer token on all /api/* routes
- Added `checkAuth()` middleware checking `Authorization: Bearer <token>`
- Applied to all routes matching `/api/*` prefix
- `/health` and `/` (HTML) remain unauthenticated
- Frontend: all `fetch()` calls now include `Authorization` header with AUTH_TOKEN

### T0.1.4 — Shell injection: getLogs() fixed
- Replaced string-interpolation `exec()` with `spawnSync()` array-form args
- Added `safeLimit` validation (1-500, coerced to integer)
- No shell expansion possible

### T0.1.5 — jspawnCron error handling
- Now returns `{ok: true, data: ...}` or `{ok: false, error: ...}`
- All callers updated to check `result.ok`
- Structured error responses with `code` field

### T0.1.6 — sessionCost dedup
- Backend already included cacheWrite — confirmed correct
- Frontend sessionCost now also includes cacheWrite: `(cw/1e6)*.10`
- Both match: input=0.30, output=1.20, cacheRead=0.06, cacheWrite=0.10

### T0.1.7 — Rate limiting: 100 req/min per IP
- In-memory `Map<ip, {count, resetAt}>` tracker
- Returns 429 with `{ok: false, error: 'Rate limit exceeded', code: 'RATE_LIMITED'}`
- Probabilistic cleanup (1% chance per request) to prevent memory leak

## Issues Found
- None critical
- Note: AUTH_TOKEN is hardcoded in both files — should move to env var in future phase
- Note: Rate limiting is in-memory only — resets on server restart (acceptable for single-server)
