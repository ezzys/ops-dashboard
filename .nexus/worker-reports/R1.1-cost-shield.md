## R1.1.3 — Cost Velocity Tracking

- **Rolling 5-min window** (`_costWindow[]`): stores `{ts, cost}` snapshots, prunes entries older than 5 min
- **Spend rate**: `deltaCost / (deltaMs / 60_000)` → USD per minute
- **Projected daily**: `spendRate × minutesRemainingInDay`
- **Budget exhaustion**: `remaining / spendRatePerMin / 60` → hours

## R1.1.4 — Budget Alert Thresholds

From `nexus-config.json → budgets.dailyUsd`:

| Threshold | Level | Action |
|-----------|-------|--------|
| 50% | `info` | WebSocket `cost-alert` broadcast |
| 80% | `warn` | WebSocket `cost-alert` broadcast |
| 95% | `critical` | WebSocket `cost-alert` broadcast |
| 100% | `critical` | WebSocket `cost-alert` broadcast (level `blocked`) |

Alert deduplication via `_alertsSent` Set — resets at day boundary. WebSocket payload:
```json
{"type":"cost-alert","level":"warn","data":{"threshold":80,"msg":"...","pct":82,"totalCost":0.82,"limit":1.00,"ts":...}}
```

## R1.1.6 — Anomaly Detection

- Maintains `_rateHistory[]` — last 12 spend rate samples (6 min of data)
- **Trigger**: `spendRate > avgRate * 3` AND `rateHistory.length >= 3` AND both > 0
- Broadcasts `cost-alert` with `level: 'warn'`, message: "Spend rate >3x rolling average — possible token spiral"
- **Re-arms** after 5 minutes via `setTimeout(_alertsSent.delete('anomaly'), 300_000)`

## R1.1.7 — Frontend Cost Shield Panel

`index.html` — `renderCost()` rewritten as full Cost Shield:

### Components
1. **Budget bars** — dual progress bars (daily + monthly), color-coded by threshold
   - `b-ok` (green) / `b-info` (blue at 50%) / `b-warn` (yellow at 80%) / `b-crit` (red pulse at 95%)
2. **Velocity gauge** — spend rate ($/hr), projected daily spend, hours until budget exhausted
   - `⚡` when spending / `💤` at zero / `🚨` on anomaly (highlighted border)
3. **Alert list** (`#cs-alert-list`) — last 5 received `cost-alert` events with level color coding
4. **Stat grid** — Total Cost, Sessions, Input/Output tokens, Cache Reads, Total Tokens
5. **Mini bar chart** — last 30 min of cost history, 30-second granularity, up to 40 CSS bars
6. **By Model table** — session count + token breakdown + cost per model
7. **By Agent/Skill table** — attribution breakdown (only shown when non-empty)
8. **Session Detail table** — per-session rows from `/api/modelusage`

### WebSocket Integration
- `connectCostWs()` connects on `boot()` to `ws://.../ws?room=cost-events&token=...`
- Auto-reconnects after 8 seconds on disconnect
- Live indicator pill in budget card header — green pulsing dot when connected
- `cost-update` → updates `S.data.costSummary`, re-renders cost tab if active
- `cost-alert` → pushes to `S.costAlerts[]`, updates `#cs-alert-list` live

## Architecture Notes

- Aggregator is intentionally **fire-and-forget** — errors are swallowed to never crash the server
- All broadcasts go through the same `broadcast(room, payload)` function from `server.js`
- Frontend falls back gracefully: if `S.data.costSummary` is null, renders from `S.data.cost` (legacy modelusage data); if both null, shows skeleton
- Alert deduplication is per-day to avoid spam across restarts within the same day
