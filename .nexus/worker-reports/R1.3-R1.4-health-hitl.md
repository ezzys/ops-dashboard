## R1.3 — Agent Health Heartbeat Grid

### R1.3.1–R1.3.2 — Health Monitor Service (`src/services/health-monitor.js`)

- Reads agent activity from `sessiondb.getRecentSessions(100)`
- Falls back to `openclaw.getStatus()` CLI when DB unavailable
- Groups sessions by agent identity (`agent_id`, `agentId`, `skill_name`, etc.)
- Extracts last-activity timestamp from any plausible session column
- Computes context% from `context_tokens / context_limit` (defaults to 200k limit)

**Status computation thresholds** (from `nexus-config.json`):
- `healthy`:  age ≤ `warnHeartbeatMs` (120s)
- `warning`:  age > 120s and ≤ `stuckMinutes * 60s` (5 min)
- `stuck`:    age > 5 min
- `offline`:  no timestamp or no session data

### R1.3.3 — Stuck Detection

- Implemented via `stuckMinutes` config (default 5 min)
- Status transitions: healthy → warning → stuck as silence grows

### R1.3.4 — Auto-Restart Config

- Per-agent override: `nexus-config.json → agentHealth.agents.<id>.autoRestart`
- Falls back to `agentHealth.mode` (default: `"alert-only"`)
- Valid values: `alert-only` / `auto-restart` / `manual`
- Displayed on each agent card in the UI

### R1.3.5 — Health Grid UI (index.html)

New **Agents** tab (9th tab, desktop + mobile):
- Summary pills: healthy/warning/stuck/offline counts + WS live indicator
- Agent cards: status icon, name, last-seen age, context%, auto-restart mode
- Status states: 🟢 Healthy, 🟡 Warning, 🔴 Stuck, ⚫ Offline
- Stuck cards pulse red (CSS animation)
- Each card has `INTERVENE` button → opens HITL slide-over

**Empty state**: "No agent sessions found" message when session DB unavailable.

### R1.3.6 — Health WebSocket Broadcasts

- Broadcasts `{type: 'health-update', data: {agents, counts, ts}}` to `health-events` room every 30s
- Client connects via `connectHealthWs()` at boot
- Live/Polling pill in UI reflects connection state
- Badge on Agents tab shows total agent count; red dot when any agents stuck

## R1.4 — HITL Intervention

### R1.4.1–R1.4.4 — Intervention Routes (`src/routes/intervention.js`)

| Route | Method | Body | Notes |
|-------|--------|------|-------|
| `/api/intervention/pause` | POST | `{sessionId, confirmed: true}` | Halts new model calls |
| `/api/intervention/resume` | POST | `{sessionId, confirmed: true}` | Unblocks paused session |
| `/api/intervention/inject` | POST | `{sessionId, message}` | Injects user message (max 4096 chars) |
| `/api/intervention/terminate` | POST | `{sessionId, confirmation: "TERMINATE-<id>"}` | Hard kill, typed confirmation |
| `/api/intervention/audit` | GET | `?limit=N` | Filtered view of intervention actions |

All routes:
- Validate sessionId with `/^[\w\-.]+$/` (no shell injection surface)
- All calls go via `spawnCli()` array-args — no shell interpolation
- All actions logged via `audit-log.js` (`session-pause`, `session-resume`, `session-inject`, `session-terminate`)

### R1.4.5 — Audit Integration

- Every intervention action audit-logged to `data/audit.db` (SQLite) or `data/audit.jsonl` fallback
- `/api/intervention/audit` filters `getRecentActions()` by intervention action names

### R1.4.6 — HITL Slide-Over UI (index.html)

Triggered by `INTERVENE` button on any agent card:
- **Session summary**: ID, status, last-seen timestamp, context%
- **Pause** action with moderate-risk styling + confirmation
- **Resume** action with safe styling
- **Inject Message** — expandable compose area with text input
- **Terminate** — expandable area requiring exact typed string `TERMINATE-<sessionId>`
- Inline result banners (✓/✗) that auto-dismiss after 8s
- Recent intervention audit log for the specific session (filtered from `/api/intervention/audit`)
- Slide-over closes on overlay click or Escape key
