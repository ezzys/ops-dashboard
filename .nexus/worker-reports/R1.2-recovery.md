## Backend Routes (pre-existing, verified)

### R1.2.1 — `POST /api/recovery/gateway-restart`
- Requires `{confirmed: true}` in body (schema-enforced)
- Async spawn of `openclaw gateway restart` with 30s timeout
- Returns `{ok, duration}` or `{ok: false, error, duration}`
- Audit-logged regardless of outcome

### R1.2.2 — `POST /api/recovery/session-clear`
- Requires `{sessionId: string, confirmed: true}`
- sessionId validated: alphanumeric+hyphens only (no shell injection surface)
- Uses `spawnCli(['session', 'clear', '--id', sessionId])` (array args, no injection)
- Returns `{ok, sessionId}` or `{ok: false, error}`
- Audit-logged

### R1.2.3 — `POST /api/recovery/agent-kill`
- Requires `{pid: integer, confirmed: true}`
- pid range: 2–4194304 (schema-enforced); refuses `process.pid` and pid 1
- SIGTERM sent immediately; SIGKILL scheduled after 5s if process still alive
- Escalation logged as separate `agent-kill-escalate` audit entry
- Returns `{ok, pid, signal}` or `{ok: false, error}`

### R1.2.4 — `POST /api/recovery/session-clear-all`
- Requires `{confirmation: 'CLEAR-ALL'}` (exact string match)
- Returns `{ok: true}` or `{ok: false, error}`
- Audit-logged

### R1.2.5 — `GET /api/recovery/audit`
- Query param: `?limit=N` (default 100, max 500)
- Returns `{ok, entries, count, ts}`

## Audit Log Service (`src/services/audit-log.js`)

- **Primary store**: SQLite (`data/audit.db`, table `audit_log`)  
  - Confirmed working: smoke-test write+read returned `backend: sqlite`
- **Fallback**: append-only JSONL (`data/audit.jsonl`) if SQLite unavailable
- Schema: `{id, ts, operator, action, target, before, after, ok, error_msg}`
- `logAction()` never throws — swallows all storage errors to protect server
- `getRecentActions(limit)` returns newest-first, normalized to JS objects

## Security Notes

- All destructive actions require explicit confirmation in the request body — schema-enforced at Fastify layer
- Session ID input validated to `[\w\-.]+` regex before any CLI call (no shell injection)
- PID validated as integer 2–4194304; server's own PID and init (1) explicitly rejected
- `CLEAR-ALL` requires exact case-sensitive string match
- Every action audit-logged — no exceptions, even failed/not-found attempts
- CLI calls use array args via `spawnCli` / `spawn` — no string interpolation into shell
