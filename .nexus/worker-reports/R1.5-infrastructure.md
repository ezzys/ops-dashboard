## WebSocket Server (R1.5.2)

- Manual WS upgrade via `ws` library (not `@fastify/websocket`) — cleaner and decoupled
- Auth required: `?token=<bearer>` query param or `Authorization` header
- Room-based subscriptions: clients connect to `/ws?room=<room>&token=<token>`
- Valid rooms: `cost-events`, `health-events`, `logs`
- Invalid room → `400 Bad Request` (socket destroyed)
- Invalid/missing token → `401 Unauthorized` (socket destroyed)
- `broadcast(room, payload)` helper exported for Phase 1 event pushes

**Tested:**
```
ws://localhost:18792/ws?room=cost-events&token=... → connected, receives {"type":"connected","room":"cost-events"}
ws://localhost:18792/ws?room=invalid&token=... → 400 (rejected)
```

## Session DB Reader (R1.5.4)

`src/sessiondb.js`:
- Searches candidate paths for `sessions.db`
- Falls back gracefully if DB not found (returns `[]`)
- Exports: `getSessions()`, `getSessionById(id)`, `getRecentSessions(limit)`, `getDbStatus()`
- Auto-discovers table and column names via `PRAGMA table_info`
- Read-only mode (`readonly: true, fileMustExist: true`)

**Note:** DB path `/Users/openclaw/.openclaw/data/sessions.db` does not yet exist. `getDbStatus()` returns `{ available: false }`. When the DB is created, the reader will auto-detect it.
