# T0.2-T0.3 Stability + Observability — Complete

## T0.2 Stability Patches

### T0.2.1 — Request timeouts
- Added 30s timeout to every request via `setTimeout` in route handler
- Server-level `headersTimeout` and `requestTimeout` set to 31s
- Returns 504 with `{ok:false, error:'Request timeout', code:'TIMEOUT'}`

### T0.2.2 — Tab timer memory leak
- Already handled correctly in existing code
- `scheduleTabRefresh()` calls `clearInterval()` before setting new timer
- Visibility change handler clears and recreates timers

### T0.2.3 — Duplicate sysmetrics fetch
- No actual duplicate found — `fetchData()` sets `S.data.sys`, subsequent `if(!S.data.sys)` check prevents re-fetch

### T0.2.4 — Circuit breaker
- Added `wrapCli()` wrapper tracking consecutive CLI failures
- After 3 failures: circuit opens, returns cached data with warning
- Auto-recovers after 60s cooldown
- Caches last-known-good responses per function

### T0.2.5 — Structured error responses
- All error responses now use `{ok: false, error: string, code: string}` format
- Error codes: `RATE_LIMITED`, `AUTH_REQUIRED`, `CLI_ERROR`, `BAD_REQUEST`, `NOT_FOUND`, `TIMEOUT`

## T0.3 Observability Foundation

### T0.3.2 — Request IDs
- Auto-generated `req-N` counter per process
