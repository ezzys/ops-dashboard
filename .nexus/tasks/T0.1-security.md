## Tasks (do ALL of them)

### T0.1.1 — XSS: Apply esc() to all user data in index.html template literals
Read index.html, find all places where user-controlled data (cron names, descriptions, session IDs, log content) is inserted into HTML via template literals. Create an `esc()` function that escapes `<`, `>`, `&`, `"`, `'` and apply it to all user data interpolations. Verify by searching for any remaining `${` that reference user data without esc().

### T0.1.2 — CORS: Replace `Access-Control-Allow-Origin: *` with same-origin
In dashboard.js, change the CORS header from `*` to check the request origin matches the server. Only allow requests from `localhost:18790` or `127.0.0.1:18790`.

### T0.1.3 — Auth: Add Bearer token check to all /api/* routes
In dashboard.js, add middleware that checks `Authorization: Bearer <token>` header against the existing TOKEN constant. Return 401 for missing/invalid tokens. The /health endpoint should remain unauthenticated.

### T0.1.4 — Shell injection: Fix getLogs() — use array-form spawn
The current `getLogs()` uses string interpolation with `limit` parameter. Refactor to use `spawnSync` with array args (like `spawnCron` already does). Validate `limit` is a positive integer.

### T0.1.5 — jspawnCron error handling
Ensure `jspawnCron` returns `{ok: false, error: message}` on CLI failure instead of throwing or returning null.

### T0.1.6 — Fix sessionCost duplication
Backend should calculate all costs including cacheWrite. Frontend should only display, never recalculate. Check index.html for any duplicate cost calculations and remove them.

### T0.1.7 — Rate limiting: 100 req/min per IP
Add in-memory rate limiting: track requests per IP, return 429 after 100 requests in a 60s window. Use a simple Map with periodic cleanup.
