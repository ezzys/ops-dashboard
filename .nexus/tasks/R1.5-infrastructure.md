# Worker Task: R1.5 — Phase 1 Infrastructure (Fastify + WebSocket + Config + SQLite)

## Project: /tmp/ops-dashboard
## You are building the backend infrastructure that Phase 1 features depend on.

## Context
The current dashboard is a single `dashboard.js` (~520 lines) using raw `http.createServer`. It needs to be upgraded to Fastify for Phase 1 features (WebSocket, typed routes, validation).

## Tasks

### R1.5.1 — Fastify scaffold
- Install fastify + dependencies: `cd /tmp/ops-dashboard && npm init -y && npm install fastify ws @fastify/cors @fastify/static better-sqlite3`
- Create `src/server.js` — Fastify server replacing dashboard.js
- Migrate ALL existing routes from dashboard.js to Fastify route format
- Keep all existing functionality working (health, data, logs, sysmetrics, modelusage, research, cron/*)
- Include all Phase 0 patches: auth check, rate limiting, circuit breaker, structured logging, request IDs

### R1.5.2 — WebSocket server
- Integrate `ws` with Fastify using `@fastify/websocket` or manual upgrade
- Support room-based subscriptions: `cost-events`, `health-events`, `logs`
- Clients subscribe via query param: `ws://localhost:18790/ws?room=cost-events`

### R1.5.3 — Config file
- Create `nexus-config.yaml` (or `nexus-config.json`) with:
  - budgets: { dailyUsd: 1.00, monthlyUsd: 30.00 }
  - thresholds: { stuckMinutes: 5, warnHeartbeatMs: 120000, errorHeartbeatMs: 30000 }
  - model prices (from existing MODEL_PRICES)
  - auth token
  - rate limiting config
  - retention days: 30
- Create `src/config.js` to load and validate it

### R1.5.4 — Session DB reader
- Create `src/sessiondb.js` using better-sqlite3
- Direct read-only access to OpenClaw's session SQLite DB
- Methods: getSessions(), getSessionById(), getRecentSessions(limit)
- Find the actual DB path from OpenClaw config or filesystem

### R1.5.5 — Shared OpenTelemetry setup
- Create `src/telemetry.js` with basic OTel setup
- Export helpers: startSpan(), endSpan(), addEvent()
- gen_ai semantic conventions for future use
