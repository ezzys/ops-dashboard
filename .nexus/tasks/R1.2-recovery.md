# Worker Task: R1.2 — One-Click Recovery Console

## Project: /tmp/ops-dashboard
## Read /tmp/ops-dashboard/.nexus/worker-reports/R1.5-infrastructure.md FIRST to understand the existing codebase.

## Context
The Fastify scaffold is done. You're building the Recovery Console for operator intervention.

## Architecture
- Backend: Fastify routes in `src/routes/`
- CLI wrapper: `src/services/openclaw.js` handles all openclaw CLI calls
- Config: `nexus-config.json`
- Auth middleware already in place

## Tasks

### R1.2.1 — Gateway restart
Create `POST /api/recovery/gateway-restart`:
- Runs `openclaw gateway restart` via spawn
- Returns `{ok: true, duration}` or `{ok: false, error}`
- Confirmation flow: frontend must send `{confirmed: true}`

### R1.2.2 — Session clear (selective)
Create `POST /api/recovery/session-clear`:
- Body: `{sessionId: string, confirmed: boolean}`
- Clears specific session
- Use openclaw CLI or direct DB access

### R1.2.3 — Agent kill by PID
Create `POST /api/recovery/agent-kill`:
- Body: `{pid: number, confirmed: boolean}`
- Sends SIGTERM to specific PID, then SIGKILL after 5s if still alive
- Returns `{ok: true, signal}` or `{ok: false, error}`

### R1.2.4 — Session clear all (nuclear)
Create `POST /api/recovery/session-clear-all`:
- Body: `{confirmation: 'CLEAR-ALL'}` (typed confirmation required)
- Clears all sessions
- Requires exact string match

### R1.2.5 — Recovery action audit log
Create `src/services/audit-log.js`:
- Every recovery action logged: `{timestamp, operator, action, target, before, after}`
- Store in SQLite table `audit_log` (or append-only JSON file if DB not available)
- Expose `GET /api/recovery/audit` to list recent actions

### R1.2.6 — Recovery console UI
Update `index.html` to add Recovery Console section:
- Grid of action cards with risk coding:
  - 🟢 Safe: session clear (single)
  - 🟡 Moderate: gateway restart, agent kill
  - 🔴 Destructive: session clear all
- Each card: icon, label, risk badge, impact description, [Cancel] [Execute] buttons
- Confirmation modals (typed for destructive actions)
- Audit log display (recent actions table)

Write summary to /tmp/ops-dashboard/.nexus/worker-reports/R1.2-recovery.md
