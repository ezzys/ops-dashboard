## R2.1 — Event Store + Session Replay

### R2.1.1 — Event log schema (SQLite)
Create `src/services/event-store.js`:
- New SQLite DB: `data/events.db` (separate from session DB for performance)
- Tables:
```sql
CREATE TABLE events (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  trace_id TEXT NOT NULL,
  span_id TEXT NOT NULL,
  parent_span_id TEXT,
  surface TEXT NOT NULL CHECK(surface IN ('cognitive','operational','contextual')),
  event_type TEXT NOT NULL,
  timestamp INTEGER NOT NULL,
  agent_id TEXT,
  model TEXT,
  data TEXT NOT NULL DEFAULT '{}',
  duration_ms INTEGER,
  status TEXT DEFAULT 'success'
);
CREATE INDEX idx_events_trace ON events(trace_id);
CREATE INDEX idx_events_surface ON events(surface);
CREATE INDEX idx_events_type ON events(event_type);
CREATE INDEX idx_events_agent ON events(agent_id);
CREATE INDEX idx_events_ts ON events(timestamp);
```
- Methods: `init()`, `writeEvent(event)`, `getEventsByTrace(traceId)`, `queryEvents(filters)`, `getRecentEvents(limit)`

## R2.2 — Three-Surface Event Types

Based on arxiv AgentTrace paper. Implement event type definitions in `src/services/event-types.js`:

### Cognitive surface
- `reasoning_step` — agent thinking/reasoning
- `tool_selected` — tool selection decision
- `confidence_score` — confidence estimate
- `context_built` — context assembly

### Operational surface
- `tool_call` — tool invocation with args
- `tool_result` — tool response
- `function_invoked` — function call
- `error` — error event
- `retry` — retry attempt

### Contextual surface
- `http_request` — HTTP call
- `db_query` — database query
- `cache_hit` / `cache_miss` — cache events
- `file_read` / `file_write` — file I/O

Export as constants + validation functions.
