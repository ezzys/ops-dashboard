## R1.3 — Agent Health Heartbeat Grid

### R1.3.1 — Heartbeat emitter
Create `src/services/health-monitor.js`:
- Track agent heartbeats: agents emit `heartbeat` events via session data
- Use `sessiondb.getRecentSessions()` or CLI `getStatus()` to get last-activity timestamps
- Compute `heartbeat_ts` per agent from session last-activity time

### R1.3.2 — Health check backend
- Compute health per agent: `stale = now - heartbeat_ts > threshold`
- Thresholds from config: `warnHeartbeatMs`, `errorHeartbeatMs`

### R1.3.3 — Stuck detection
- If no activity for > `stuckMinutes` (default 5min from config), mark as stuck
- Activity = any CLI call, tool use, or message

### R1.3.4 — Auto-restart config
- Per-agent configurable: alert-only / auto-restart / manual
- Config stored in nexus-config.json under `agentHealth`

### R1.3.5 — Health grid UI
Add to `index.html` — Agent Health Grid tab/section:
- Cards per agent: name, status (healthy/stuck/warning/offline), last seen, context %, auto-restart toggle
- Health states: 🟢 Healthy, 🟡 Warning, 🔴 Stuck, ⚫ Offline
- WebSocket subscription to `health-events` room for real-time updates

### R1.3.6 — Health WebSocket broadcasts
- Broadcast to `health-events` room every 30s with agent health status
- Payload: `{type: 'health-update', data: {agents: [{id, status, lastSeen, contextPct}]}}`

## R1.4 — HITL (Human-in-the-Loop) Intervention

### R1.4.1 — Session pause
Create routes in `src/routes/intervention.js`:
- `POST /api/intervention/pause` — body: `{sessionId, confirmed}`
- Signal gateway to pause session (no new model calls)

### R1.4.2 — Session resume
- `POST /api/intervention/resume` — body: `{sessionId, confirmed}`

### R1.4.3 — Message injection
- `POST /api/intervention/inject` — body: `{sessionId, message}`
- Inject user message into running session

### R1.4.4 — Session termination
- `POST /api/intervention/terminate` — body: `{sessionId, confirmation: 'TERMINATE-<session-id>'}`
- Hard kill with typed confirmation

### R1.4.5 — Intervention audit log
- All HITL actions logged via existing `src/services/audit-log.js`

### R1.4.6 — HITL UI
Add to `index.html` — on active session cards:
- "INTERVENE" button opens slide-over panel
- Options: Pause, Resume, Inject Message, Terminate
- Each has confirmation flow
- Shows session summary before action
