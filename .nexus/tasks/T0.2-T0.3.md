# T0.2 Stability + T0.3 Observability Patches

## T0.2.1 — Request timeouts (30s)
Add request timeout to the HTTP server. All connections close after 30s.

## T0.2.2 — Tab timer memory leak
Clear all interval timers on tab switch (the `scheduleTabRefresh` / `clearTimers` flow).

## T0.2.3 — Duplicate sysmetrics fetch
Check for duplicate `/api/sysmetrics` calls in the frontend.

## T0.2.4 — Circuit breaker
If `openclaw` CLI fails 3x consecutively, return cached data + warning until recovery.

## T0.2.5 — Structured error responses
All API errors return `{ok: false, error: string, code: string}` — already partially done in T0.1.

## T0.3.1 — Structured logging (pino)
Replace all console.log/error with pino JSON logger.

## T0.3.2 — Request ID
Add X-Request-ID to all log entries.

## T0.3.3 — OpenTelemetry spans
Add basic OTel gen_ai spans to CLI exec calls.

## T0.3.4 — /health/detailed endpoint
Returns CLI reachability, disk space, memory pressure.

## T0.3.5 — Data retention policy
TTL-based event pruning for future event store.
