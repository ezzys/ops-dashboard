## Tasks

### R1.1.1 ‚Äî Cost aggregation service
Create `src/services/cost-aggregator.js`:
- Periodic cost aggregation (every 30s): read recent sessions, compute per-session/per-model costs
- Rolling 5-minute spend rate tracking
- Expose: `getCostSummary()`, `getSpendRate()`, `getBudgetStatus()`

### R1.1.2 ‚Äî Real-time cost WebSocket pushes
In cost aggregation loop:
- On every aggregation cycle, broadcast to `cost-events` room via WebSocket
- Payload: `{type: 'cost-update', data: {totalCost, spendRate, byModel, budgetStatus}}`

### R1.1.3 ‚Äî Cost velocity tracking
- Rolling 5-minute window: `spendRate = costLast5min / 5`
- Projected daily cost: `projected = spendRate * hoursRemainingInDay`
- Budget exhaustion time: `hoursLeft = budgetRemaining / currentSpendRate`

### R1.1.4 ‚Äî Budget alert thresholds
From config `budgets.dailyUsd`:
- 50% ‚Üí informational
- 80% ‚Üí warning + Slack alert
- 95% ‚Üí modal confirmation required
- 100% ‚Üí block new sessions
Broadcast alerts via WebSocket as `{type: 'cost-alert', level: 'warn'|'critical', data}`

### R1.1.5 ‚Äî Per-agent/skill cost attribution
Track which agent/skill drives spend using session metadata (agent_id, skill_name when available)

### R1.1.6 ‚Äî Anomaly detection
Alert when cost rate >3x rolling average (token spiral signal)

### R1.1.7 ‚Äî Frontend Cost Shield panel
Update `index.html` to add a Cost Shield section:
- Stacked area chart placeholder (daily cost over time ‚Äî use simple CSS bars for now, Recharts later)
- Spend velocity gauge
- Alert badges at threshold levels
- Budget progress bar (daily + monthly)
- WebSocket listener for real-time updates
